概述

几个最常用的二分查找场景：寻找一个数、寻找左侧边界、寻找右侧边界。

注意right的初始化值，还有while的循环条件，以及左右指针的更新值（小心死循环），这三点不同的人可能会有不同的写法，选一种自己最习惯的就好。

需要注意的几点：

  • mid写成int mid = left + (right - left) / 2，防止right+left时整形溢出

  • 当搜索区间只有两个数时，当int mid = left + (right - left) / 2，mid取到的是左中位数，此时如果有left=mid+1（即能排除左边界）没问题，如果是left=mid则会陷入死循环。

  • while循环的条件：

    • while(left <= right) 的终止条件是left>right，即 left == right + 1，搜索区间为空，while 循环终止，返回 -1。
    • while(left < right) 的终止条件是 left == right，区间左右边界相等（即收缩成 1 个数），但此时 while 循环终止了。此时：

    1、如果要找的数一定在搜索区间里出现，那么可以放心地返回 left或者 right，而无需再做判断；

    2、如果要找的数不一定在搜索区间里出现，那么只要在退出循环以后，再对nums[left]单独作一次判断，看它是不是你要找的数即可，这一步操作常常叫做“后处理”。

    建议while循环的条件写成while(left < right),这样跳出循环是left==right，不用再想返回那个边界，记得后处理就好。

  • right = nums.length时，返回的索引可能在数组索引外， right= nums.length - 1，返回索引在数组索引内。
