ch01-07

需求：定义一个hello，world程序。

步骤

  1. 通过class关键字定义一个类，代码都写在该类中。
  2. 为了保证该类独立运行，定义主函数
  3. 保存一个扩展名为java的文件
  4. 在dos控制台中通过javac工具对java文件进行编译
  5. 通过java命令对生成的class文件进行编译

main函数保证该类的独立运行，它会被JVM调用，主函数是程序的运行起始点，不需要类运行可以不写主函数

class：关键字，小写，用于定义类，类名定义，首字母都大写：XxxxYyyyZzzz

多层注释不可以嵌套

java严格区分大小写

8个二进制位表示一个字节（最大255）

三个二进制位表示一位八进制；四个二进制位表示一个十六进制

负数的二进制：正数的二进制取反（0变1,1变0）加一，负数的最高位都是1（取反取来的）
6=110

一个数占四个字节（32二进制位）

0000-0000 0000-0000 0000-0000 0000-0110

1111-1111 1111-1111 1111-1111 1111-1001

+0000-0000 0000-0000 0000-0000 0000-0001

=1111-1111 1111-1111 111-1111 1111-1010=-6

变量：将不确定的数据进行存储

java是强类型语言，对每种数据都定义明确的具体数据类型，在内存中分配了不同大小的内存空间
小数默认double，
float f=2.3f；
byte是一个八位（字节）
int，float是四个八位
整数默认int

a 97
b 98
A 65
字符1： 49
char类型可以进行运算
System.out.println((char)('a'+1));   //b

1%-5 //1
-1%5 //-1(负数只看左边)

b=a++先赋值给b，再a自增 
b=++a先自增

字符串数据和任何数据都可以+连接，最后都会变成字符串

转义字符
\t 制表符，相当于tab键
\r 按下回车键。Windows系统中，回车由\r\n表示
\b 退格，相当于backspace
\n 换行

面试题：
short s=4;
s=s+5;//编译失败，两次运算，先加法再赋值，5是int，加完是int，赋给s会精度丢失
s+=5;//成功，只做赋值运算，有自动转换动作（和s=4一样，在short范围内会自动转换）

异或^：两边不同是true，两边相同是false，一个数异或同一个数两次，还是那个数（数据加密）

位运算（比直接乘高效）：
左移右移：
左移<<：乘以2 的移动位数次幂
右移>>：除以
3<<2=3*2^2

三元运算符：简化if else语句

定义函数可以放上面或者放下面，但是都从主函数开始执行
如何定义一个函数？

  1. 函数是一个独立的功能，该功能的运算结果是什么先明确（明确返回值类型）
  2. 明确是否需要未知的内容参与运算（明确参数类型和个数）

重载：

当定义的功能相同，参与运算的位置内容不同，那么就定义一个函数名称，方便阅读，通过参数列表的不同来区分多个同名函数

面向对象:

找对象，建对象，用对象，维护对象的关系

类中的成员（成员变量，成员函数）

静态代码块：

static{ }

只执行一次，给类初始化
构造代码块，优先于构造函数（给对应对象初始化）执行
给所有对象进行统一初始化
构造函数：给对应对象初始化

this：

哪个对象在调用this所在的函数，this就代表哪个对象

this的应用：本类功能用到的本类对象，用this表示

this语句用于构造函数间相互调用
this语句只能定义在构造函数的第一行

static特点

  1. 随着类的加载而加载，类的消失而消失
  2. 优先于对象存在（静态先存在，对象后存在）
  3. 被所有对象共享
  4. 可以直接被类名调用

实例变量和类变量区别：

  1. 存放位置

    类变量随着类的加载而存于方法区中;
    实例变量随着对象的建立二存在于堆内存中

  2. 生命周期
    类变量生命周期最长

静态使用注意事项：

  1. 静态方法只能访问静态成员
    非静态方法既可以访问静态也可以访问非静态
  2. 静态方法中不可以定义this，super关键字
    因为静态优先于对象存在
  3. 主函数是静态的

静态有利有弊：

利：对对象的共享数据进行单独空间的存储，节省空
可以被类名直接调用

弊：生命周期过长；访问出现局限性（静态只能访问静态 ）

何时使用静态：

从两方面下手:

因为静态修饰的内容有成员变量和函数

什么时候定义静态变量呢？
当对象中出现共享数据是，该数据被静态所修饰
对象中的特有数据要定义成非静态存在于堆内存

什么时候定义静态函数呢？
当功能内部没有访问到非静态数据时，那么该功能可以定义成静态的

main函数：

  • public：代表该函数访问权限是最大的
  • void：主函数没有具体的返回值
  • main：不是关键字，但是是一个特殊的单词，可以被jvm识别
  • （String[] args）：函数的参数，字符串类型的数组

一个类中会有一个默认的空参数构造函数，其权限随着类的变化而变化

对象初始化过程：

该句话做了什么事情？

  1. 因为new用到了Person.class文件，所以先找到Person.class文件加载到文件中
  2. 执行类中的static代码块，给类初始化
  3. 在堆内存中开辟空间，分配内存地址
  4. 在堆内存中建立对象的特有属性，并进行默认初始化（null，0）
  5. 对属性进行显示初始化
  6. 对对象进行构造代码块初始化
  7. 对对象进行构造函数初始化
  8. 将内存地址赋给栈内存的p变量

设计模式：

java中有23种设计模式

单例设计模式：解决一个类在内存只存在一个对象

饿汉式：Single类一进内存，就已经创建好了对象（定义单例，建议用饿汉式）

懒汉式：Single类进内存，对象还没有存在，调用getInstance方法时才在堆内存里加载对象，把地址值给s，s指向对象，也叫对象的延迟加载

java只支持单继承，不支持多继承（一个孩子一个父亲）

java支持多层继承，也就是继承体系。想要使用体系，：查阅父类功能，创建子类对象使用功能

类中成员：

变量
函数
构造函数

在对子类对象进行初始化时，父类的构造函数也会运行

this（）和super（）都是只能出现在第一行（初始化要先做），所以二者不能同时出现

子类的实例化过程：

final：修饰符

  1. 可以修饰类，函数，变量
  2. 被final修饰的类不能被继承，为了避免被子类继承，被子类复写功能
  3. 被final修饰的变量是一个常量（所有字母大写，_连接），只能被赋值一次
  4. 被final修饰方法不可以被重写

内部类

内部类的访问规则：

  1. 内部类可以直接访问外部类中的成员，包括私有

    之所以可以直接访问，因为内部类中持有一个外部类的引用 外部类名.this
  2. 外部类要访问内部类，必须建立内部类对象

内部类被定义在局部时：

  1. 不可以被成员修饰符修饰
  2. 可以访问外部类中的成员，但是不可以访问它所在局部中的变量，只能访问被final修饰的局部变量

匿名内部类：

  1. 内部类的简写格式
  2. 定义匿名内部类的前提：
    内部类必须是继承一个类或者实现接口
  3. 格式：new 父类或者接口（）{定义子类的内容}
  4. 其实匿名内部类就是一个匿名子类对象（带内容的对象）
  5. 类中定义的方法最好不要超过三个
